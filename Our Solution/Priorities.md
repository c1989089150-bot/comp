# Development Priorities
Before doing anything:
- Figma prototype our solution - this should show how the user will navigate our product 

## High Priority:
1. Retrieve Info Dataset:
Retrieve all the course information for UQ that will be used in our solution: restricted to 1st year computer science courses. This will serve as the example set of data for our solution

2. Core User Flow:
The main priority is to create a functional user journey that allows the user to use the main parts of the tool. This includes all the pages laid out prior. Fundamentally, users should be able to: Select or search for a degree, view relevant courses in that degree, and view information on these courses

## Medium Priority
1. Filter and Sorting on Course Screen:
Being able to sort by semester, or course difficulty, or any other metrics would ideally be included in our final solution, but is not necessary to meet product requirements.

2. Extra International Support:
Having options for international students to get increased guidance throughout our product is relevant to engage with our target customer groups, although it is not strictly necessary. Our product is being designed to be simple regardless of providing this additional guidance, so this is a medium priority task.

3. Additional Course Information 
More detailed assessment information or descriptions of difficulty are important when viewing courses, but our solution focuses on showing prerequisites for selected courses. Therefore, this information should be secondary to this.


## Low Priority
1. Tree Diagram/Graphical Representation:
Visual diagrams representing how courses flow into each other would be useful, but is a time-consuming task. It is also not strictly required, assuming prerequisite information is still accessible.
