# Our Solution Features
The Following Features will be included in our initial prototype:
Note: For our basic prototype, we will be using first-year computer science courses as our foundation
## 1. Home Screen
    - Some basic information about our tool
    - Our product name
    - Some 'start' button of sorts, that begins the process of selecting the degree and course you want more information on 
    - MAYBE: Tick box button for international students, which can provide them more guidance than the normal user

## 2. Degree Selection Screen
    - Will provide a list or search bar allowing for people to search by degree code or degree name
    - Two buttons located at the bottom of the screen, will allow them to navigate back or to confirm their selection

## 3. Course Screen
    - From here, our solution will provide a list of the courses the student should eventually enrol in, or are part of their degree
    - Search bar/sort bar will allow users to find courses depending on the semester, prerequisites, etc.
    - Users can click on a course, and it will navigate them to a page that allows them to view various course details such as; prerequisites, basic information about the course, difficulty, assessment

## 4. Course Information Screen
    - When a user clicks on a course, they are directed to a page that contains information on this course including:
        1. Prerequisites
        2. Course Overview
        3. Assessment Types
        4. Difficulty
        5. Course Staff

    - SUGGESTION: Krish suggested that we could create a tree diagram of sorts to show how the courses flow into each other? Or some graphical representation of course sequence that is less text-based
